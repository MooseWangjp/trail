# 垃圾回收

## 概念

垃圾回收（Garbage Collection，GC）是自动内存管理机制，用于自动释放不再使用的内存，放置内存泄漏。

JavaScript 是一种自动管理内存的语言，开发者不需要手动分配和释放内存。

```javascript
// 内存自动分配
let obj = { name: "test" }; // 自动分配内存

// 内存自动回收
obj = null; // 当对象不再被引用时，GC会自动回收
```

关于垃圾回收机制，还有一个特别重要的概念是“标记清除”，我们来看看标记清除算法的实现原理，理解一下垃圾回收是如何实现的。

## 标记清除算法原理

核心思想是将内存管理分为两个阶段：标记阶段和清除阶段

标记阶段是从“Root（根：根是垃圾回收的起点，始终被认为是“活跃”的）”出发，递归标记可达（活跃的）对象，不活跃的我们归类为垃圾。

清除阶段：回收未被标记的对象内存，内存重用。

这个Root有下面几种情况：

```javascript
// 1. 全局对象
window(浏览器) / global(Node.js);

// 2. 当前执行栈中的变量
function foo() {
  let local = { a: 1 }; // local 是根
}

// 3. 当前被引用的对象
let active = { data: "important" }; // active 是根
```

### 标记过程

```
// 初始状态：所有对象都标记为"不可达"
┌──────────────┐
│   内存堆      │
│  ┌────────┐  │
│  │  obj1  │ ❌ (未标记)
│  │  obj2  │ ❌ (未标记)
│  │  obj3  │ ❌ (未标记)
│  └────────┘  │
└──────────────┘

// 从根开始遍历，标记可达对象
let root = { name: 'root' };  // root 是根
root.child = obj1;            // obj1 可达
obj1.next = obj2;             // obj2 可达
// obj3 没有被引用

┌──────────────┐
│   内存堆      │
│  ┌────────┐  │
│  │  obj1  │ ✅ (已标记 - 可达)
│  │  obj2  │ ✅ (已标记 - 可达)
│  │  obj3  │ ❌ (未标记 - 垃圾)
│  └────────┘  │
└──────────────┘

```

### 清除过程

```
// 清除阶段：回收所有未标记的对象
function sweep() {
    for (let obj of heap) {
        if (!obj.marked) {
            // 释放内存
            freeMemory(obj);
        } else {
            // 重置标记位，为下次GC做准备
            obj.marked = false;
        }
    }
}
```

1. 分代回收

```
// 分代假设：大部分对象生命周期很短

// 新生代（1-8MB）
function shortLived() {
    let temp = { data: 'temporary' };  // 存放在新生代
    // 函数执行完后，temp 很快死亡
}

// 老生代
let config = { settings: 'persistent' };  // 存放在老生代
// 长期存活的对象

```

新生代 -- Scavenge算法

- Cheney算法将内存分为两半
- From空间复制到To空间
- 存活对象复制后，From空间整体清空
- 优点是：无碎片 缺点：只能使用一半内存

2. 并发标记
   主线程继续执行JavaScript任务，后台线程进行GC标记

3. 增量标记
   将GC工作分成小片段，穿插在JS执行中，避免长时间暂停，比如:

   执行JS(5ms) --> 标记（3ms）--> JS(5ms) --> 标记（3ms）-->...

4. 标记整理
   解决内存碎片问题，将存活对象向一端移动，整理出连续内存

   整理前: [存活][空][空][存活][空][存活]

   整理后: [存活][存活][存活][空][空][空]

搞清楚了垃圾回收相关的机制，我们来看看在编码过程中如何来配合这个机制，让我们更好地利用有限的内存。

## 实践

1. 及时解除引用

```javascript
function processData() {
  let data = getLargeData();
  // 处理数据
  processDataLogic(data);
  data = null; // 显式解除引用
}
```

2. 避免全局变量，这会导致永远不会回收

```javascript
// 全局变量
var data = {
  /* large data */
};

// 使用局部作用域
function init() {
  let data = {
    /* large data */
  };
  // ...
}
```

3. 及时清理事件监听器

```javascript
let element = document.getElementById("btn");
let handler = () => {
  /* ... */
};
element.addEventListener("click", handler);
// ...
element.removeEventListener("click", handler); // 记得移除
```
